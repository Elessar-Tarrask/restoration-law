# Визуальная схема процесса восстановления платежеспособности

## Диаграмма процесса (Mermaid)

```mermaid
flowchart TD
    Start([Клиент обращается в ТОО]) --> Consult[ЭТАП 1: Первичная консультация<br/>Срок: 1-2 дня]
    
    Consult --> CheckCriteria{Соответствует<br/>критериям?}
    CheckCriteria -->|Нет| Reject([Отказ в услуге])
    CheckCriteria -->|Да| Contract[Заключение договора]
    
    Contract --> CollectDocs[ЭТАП 2: Сбор документов<br/>Срок: 5-10 дней]
    CollectDocs --> PrepareApp[ЭТАП 3: Подготовка заявления<br/>Срок: 2-3 дня]
    PrepareApp --> SubmitApp[ЭТАП 4: Подача в суд<br/>Срок: 1 день]
    
    SubmitApp --> CourtReview[ЭТАП 5: Рассмотрение судом<br/>Срок: 10 рабочих дней]
    CourtReview --> CourtDecision{Решение суда}
    
    CourtDecision -->|Отказ| Appeal{Обжалование?}
    Appeal -->|Да| CourtReview
    Appeal -->|Нет| EndReject([Процедура прекращена])
    
    CourtDecision -->|Возбуждение дела| AppointManager[ЭТАП 6: Назначение<br/>финансового управляющего<br/>Срок: 3-5 дней]
    
    AppointManager --> DevelopPlan[ЭТАП 7: Разработка плана<br/>восстановления<br/>Срок: 15-30 дней]
    
    DevelopPlan --> ApprovePlan[ЭТАП 8: Утверждение плана<br/>судом<br/>Срок: 10-15 дней]
    
    ApprovePlan --> PlanDecision{План утвержден?}
    PlanDecision -->|Нет| ModifyPlan[Изменение плана]
    ModifyPlan --> DevelopPlan
    PlanDecision -->|Да| ExecutePlan[ЭТАП 9: Исполнение плана<br/>Срок: До 5 лет]
    
    ExecutePlan --> PaymentCheck{Платежи<br/>вовремя?}
    PaymentCheck -->|Нет| Violation[Нарушение плана]
    Violation --> ViolationDecision{Решение суда}
    ViolationDecision -->|Изменение плана| ModifyPlan
    ViolationDecision -->|Банкротство| Bankruptcy([Процедура банкротства])
    
    PaymentCheck -->|Да| AllPaid{Все долги<br/>погашены?}
    AllPaid -->|Нет| ExecutePlan
    AllPaid -->|Да| Complete[ЭТАП 10: Завершение<br/>процедуры<br/>Срок: 10-15 дней]
    
    Complete --> Success([Процедура успешно<br/>завершена!])
    
    style Start fill:#e1f5ff
    style Success fill:#d4edda
    style EndReject fill:#f8d7da
    style Bankruptcy fill:#f8d7da
    style Consult fill:#fff3cd
    style CollectDocs fill:#fff3cd
    style PrepareApp fill:#fff3cd
    style SubmitApp fill:#fff3cd
    style CourtReview fill:#fff3cd
    style AppointManager fill:#fff3cd
    style DevelopPlan fill:#fff3cd
    style ApprovePlan fill:#fff3cd
    style ExecutePlan fill:#d1ecf1
    style Complete fill:#d4edda
```

## Схема ролей и взаимодействий

```mermaid
graph TB
    subgraph "Участники процесса"
        Client[КЛИЕНТ<br/>Должник<br/>━━━━━━━━<br/>Обязанности:<br/>• Предоставить документы<br/>• Исполнять план<br/>• Информировать об изменениях<br/>━━━━━━━━<br/>Права:<br/>• Получать информацию<br/>• Участвовать в заседаниях<br/>• Обжаловать решения]
        
        TOO[ТОО<br/>Консультант<br/>━━━━━━━━<br/>Обязанности:<br/>• Консультировать<br/>• Подготавливать документы<br/>• Представлять интересы<br/>• Отслеживать процесс<br/>━━━━━━━━<br/>Права:<br/>• Получать информацию<br/>• Представлять клиента<br/>• Получать вознаграждение]
        
        Court[СУД<br/>━━━━━━━━<br/>Обязанности:<br/>• Рассмотреть заявление<br/>• Назначить управляющего<br/>• Утвердить план<br/>• Контролировать исполнение<br/>━━━━━━━━<br/>Права:<br/>• Запрашивать документы<br/>• Отклонить заявление<br/>• Принять меры]
        
        Manager[ФИНАНСОВЫЙ<br/>УПРАВЛЯЮЩИЙ<br/>━━━━━━━━<br/>Обязанности:<br/>• Разработать план<br/>• Контролировать исполнение<br/>• Распределять средства<br/>• Отчитываться в суд<br/>━━━━━━━━<br/>Права:<br/>• Запрашивать информацию<br/>• Предлагать изменения<br/>• Получать вознаграждение]
        
        Creditors[КРЕДИТОРЫ<br/>━━━━━━━━<br/>Обязанности:<br/>• Предоставить информацию<br/>• Соблюдать процедуру<br/>━━━━━━━━<br/>Права:<br/>• Получать информацию<br/>• Участвовать в заседаниях<br/>• Получать платежи<br/>• Обжаловать план]
    end
    
    Client <-->|Консультирует,<br/>помогает| TOO
    TOO -->|Подает заявление| Court
    Client -->|Подает заявление<br/>через ТОО| Court
    Court -->|Назначает| Manager
    Manager <-->|Взаимодействует| Client
    Manager <-->|Консультирует| TOO
    Manager -->|Отчитывается| Court
    Manager <-->|Распределяет средства| Creditors
    Court <-->|Рассматривает<br/>жалобы| Creditors
    Client -->|Платит по плану| Creditors
    
    style Client fill:#e1f5ff
    style TOO fill:#fff3cd
    style Court fill:#d1ecf1
    style Manager fill:#d4edda
    style Creditors fill:#f8d7da
```

## Схема принятия решений

```mermaid
flowchart TD
    Start([Начало процедуры]) --> Criteria1{Стабильный доход?}
    Criteria1 -->|Нет| Alt1[Рассмотреть внесудебное<br/>или судебное банкротство]
    Criteria1 -->|Да| Criteria2{Можно погасить<br/>в течение 5 лет?}
    
    Criteria2 -->|Нет| Alt1
    Criteria2 -->|Да| Criteria3{Есть попытки<br/>урегулирования?}
    
    Criteria3 -->|Нет| NeedDocs[Необходимо провести<br/>процедуру урегулирования]
    NeedDocs --> Criteria3
    Criteria3 -->|Да| Criteria4{Нет признаков<br/>злостного уклонения?}
    
    Criteria4 -->|Нет| Reject([Отказ в процедуре])
    Criteria4 -->|Да| Proceed[Можно приступать<br/>к процедуре]
    
    Proceed --> Submit[Подача заявления]
    Submit --> CourtCheck{Суд проверил<br/>документы}
    
    CourtCheck -->|Неполные| AddDocs[Дополнить документы]
    AddDocs --> Submit
    CourtCheck -->|Полные| Appoint[Назначение управляющего]
    
    Appoint --> Develop[Разработка плана]
    Develop --> PlanCheck{План реалистичен?}
    
    PlanCheck -->|Нет| Modify[Изменить план]
    Modify --> Develop
    PlanCheck -->|Да| Approve[Утверждение судом]
    
    Approve --> Execute[Исполнение плана]
    Execute --> PaymentCheck{Платежи<br/>вовремя?}
    
    PaymentCheck -->|Нет| Problem{Временные<br/>трудности?}
    Problem -->|Да| ChangePlan[Изменить план]
    ChangePlan --> Approve
    Problem -->|Нет| Bankruptcy([Банкротство])
    
    PaymentCheck -->|Да| AllPaid{Все погашено?}
    AllPaid -->|Нет| Execute
    AllPaid -->|Да| Success([Успешное завершение])
    
    style Start fill:#e1f5ff

    style Reject fill:#f8d7da
    style Bankruptcy fill:#f8d7da
    style Proceed fill:#fff3cd
    style Submit fill:#fff3cd
```

## Схема взаимодействия на этапе разработки плана

```mermaid
sequenceDiagram
    participant C as Клиент
    participant T as ТОО
    participant M as Финансовый<br/>управляющий
    participant Cr as Кредиторы
    participant S as Суд
    
    Note over C,S: ЭТАП 7: Разработка плана восстановления
    
    C->>T: Предоставляет информацию о доходах и расходах
    T->>M: Передает информацию о должнике
    M->>C: Запрашивает дополнительные документы
    C->>M: Предоставляет документы
    
    M->>M: Анализирует финансовое положение
    M->>M: Рассчитывает возможности погашения
    
    M->>C: Предлагает проект плана
    C->>T: Консультируется с ТОО
    T->>C: Дает рекомендации
    C->>M: Согласовывает план или предлагает изменения
    
    M->>Cr: Уведомляет о проекте плана
    Cr->>M: Предоставляют мнения (опционально)
    
    M->>S: Представляет план в суд
    S->>M: Может запросить изменения
    M->>S: Представляет окончательный план
    
    S->>C: Уведомляет о заседании
    S->>Cr: Уведомляет о заседании
    
    S->>S: Рассматривает план
    S->>C: Выносит определение об утверждении
    
    Note over C,S: План утвержден, начинается исполнение
```

## Схема контроля исполнения плана

```mermaid
flowchart TD
    Start([План утвержден судом]) --> Monthly[Ежемесячный контроль]
    
    Monthly --> CheckPayment{Платеж<br/>внесен вовремя?}
    
    CheckPayment -->|Да| CheckAmount{Сумма<br/>правильная?}
    CheckAmount -->|Нет| Notify[Уведомить должника]
    Notify --> CheckPayment
    CheckAmount -->|Да| CheckIncome{Доход<br/>стабилен?}
    
    CheckIncome -->|Нет| Report[Сообщить в суд]
    Report --> CourtDecision{Решение суда}
    CourtDecision -->|Изменить план| Modify[Изменение плана]
    CourtDecision -->|Продолжить| CheckIncome
    CourtDecision -->|Банкротство| Bankruptcy([Банкротство])
    
    CheckIncome -->|Да| Progress{Прогресс<br/>по плану?}
    Progress -->|Да| NextMonth[Следующий месяц]
    NextMonth --> Monthly
    
    CheckPayment -->|Нет| Warning[Предупреждение<br/>должнику]
    Warning --> Wait{Платеж в течение<br/>следующих 7 дней?}
    Wait -->|Да| CheckAmount
    Wait -->|Нет| Report
    
    Progress -->|Нет| Investigate[Выяснить причину]
    Investigate --> Reason{Причина}
    Reason -->|Временные трудности| Support[Поддержка,<br/>возможно изменение плана]
    Reason -->|Злостное уклонение| Report
    
    Support --> Modify
    Modify --> Monthly
    
    Monthly --> AllPaid{Все долги<br/>погашены?}
    AllPaid -->|Нет| Monthly
    AllPaid -->|Да| FinalReport[Финальный отчет<br/>управляющего]
    FinalReport --> CourtApproval[Утверждение судом]
    CourtApproval --> Complete([Процедура завершена])
    
    style Start fill:#e1f5ff
    style Complete fill:#d4edda
    style Bankruptcy fill:#f8d7da
    style Monthly fill:#fff3cd
    style CheckPayment fill:#d1ecf1
    style CheckIncome fill:#d1ecf1
    style Report fill:#f8d7da
```

## Как использовать эти схемы

1. **Диаграмма процесса** - показывает полный флоу от начала до конца процедуры
2. **Схема ролей** - описывает всех участников и их взаимодействие
3. **Схема документооборота** - показывает, какие документы передаются между участниками
4. **Временная шкала** - визуализирует сроки каждого этапа
5. **Схема принятия решений** - помогает определить, подходит ли процедура клиенту
6. **Схема взаимодействия** - детализирует процесс разработки плана
7. **Схема контроля** - показывает, как контролируется исполнение плана

Эти схемы можно использовать:
- Для презентаций клиентам
- Для обучения сотрудников
- Для создания инфографики
- Для веб-сайта компании
- Для печатных материалов
